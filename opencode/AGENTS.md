## commits
commits should follow the conventional commits guidelines (https://www.conventionalcommits.org/en/v1.0.0/)

## reviews
after asked to review the changes maded, when the review is done, you should generate a pr summary

## pr summary 
when asked to generate a pr summary, it should have the following format:
- three topics
 - ## why
 - ## changes
 - ## validation
- in markdown format

it should be sent via chat. dont generate a new file.
